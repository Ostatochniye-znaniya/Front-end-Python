# Front-end (Python) — Проверка остаточных знаний

Учебный проект по дисциплине **«Проектная деятельность»**.  
Frontend-часть системы автоматизации процессов проведения проверки остаточных знаний обучающихся в университете.

Реализация охватывает подпункты **2.5–2.6 ТЗ** и выполнена в виде веб-приложения на **Python (Flask)** с использованием HTML/CSS и клиентской логики на JavaScript.

---

## 📌 Назначение проекта

Система предназначена для:

- выбора учебных групп и дисциплин для проведения проверки остаточных знаний;
- назначения ответственных преподавателей, даты и времени проведения;
- оформления заявок на проведение проверки силами кафедр другого факультета;
- обработки входящих заявок от других подразделений;
- формирования графика проверок;
- экспорта графика проверок в CSV;
- клиентской валидации форм.

Проект реализован с использованием **мок-данных** и не требует подключения к реальной БД или API.

---

## 🧱 Архитектура проекта

Проект использует классическую структуру Flask-приложения

---

## 🧩 Реализованные компоненты

### 1. Выбор групп и дисциплин (п. 2.5.1)

Страница для заведующего кафедрой:

- таблица учебных групп;
- выбор участия группы в тестировании;
- выбор двух дисциплин;
- назначение преподавателя;
- выбор даты и времени проведения;
- клиентская валидация обязательных полей.

URL:  
`/groups-selection` (реализовано через маршрут в `app.py`)

---

### 2. Заявка на проведение проверки силами другого факультета (п. 2.5.2)

Форма создания заявки:

- выбор учебной группы;
- выбор дисциплины;
- выбор факультета-исполнителя;
- комментарий;
- клиентская валидация обязательных полей.

URL:  
`/foreign-request`

---

### 3. Обработка входящих заявок (п. 2.6)

Интерфейс для подразделения-исполнителя:

- список входящих заявок;
- просмотр деталей заявки;
- назначение преподавателя;
- указание даты и времени;
- изменение статуса заявки.

URL:  
`/incoming-requests`

---

### 4. График проверок остаточных знаний (п. 2.6)

Табличный график проверок:

- группа;
- дисциплина;
- преподаватель;
- дата и время;
- статус проверки.

Поддерживаются фильтры:

- по группе;
- по дисциплине;
- по преподавателю.

URL:  
`/schedule`

---

### 5. Экспорт графика проверок в CSV

На странице графика реализована кнопка:

**«Выгрузить в CSV»**

Функциональность:

- экспортирует текущие (отфильтрованные) данные таблицы;
- формирует CSV-файл на стороне клиента;
- файл открывается в Excel без дополнительной обработки.

---

### 6. Клиентская валидация форм

Реализована проверка:

- если группа участвует в тестировании → обязательно выбраны 2 дисциплины;
- если назначен преподаватель → обязательны дата и время;
- в заявке на чужой факультет обязательны группа, дисциплина и подразделение;
- ошибки отображаются пользователю в интерфейсе.

## 🚀 Запуск проекта

### Требования

- Python 3.9+
- pip

### Установка и запуск

```bash
cd project_root
pip install flask
python -m venv venv
```

Для Windows:
``` 
venv\Scripts\activate
```
Для Linux:
```
source venv/bin/activate
```

```bash
pip install -r requirements.txt
python app.py
```
После запуска приложение будет доступно по адресу:

http://127.0.0.1:5000
